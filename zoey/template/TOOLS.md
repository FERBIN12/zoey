# TOOLS.md — Local notes

> Skills define _how_ tools work. This file is for _your_ specifics — the environment-unique stuff.
> Keeping it separate means you can update shared skills without losing your notes, and share skills without leaking your infrastructure.

## What goes here

- Camera names and locations
- SSH hosts and aliases
- Preferred TTS voices
- Speaker / room names
- Device nicknames
- Anything environment-specific

## Example

```markdown
### Cameras
- living-room → main area, 180° wide angle
- front-door → entrance, motion-triggered

### SSH
- home-server → 10.0.0.10, user: admin

### TTS
- Preferred voice: "Nova" (warm)
- Default speaker: kitchen
```

> ⚠️ Real values here are infrastructure details. Don't commit a filled-in copy to a public repo.
