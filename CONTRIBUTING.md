# Contributing

Zoey is a small, opinionated starter kit, so the bar for changes is "does this make the pattern clearer
or safer for someone building their own agent?"

## Good contributions

- Clarity fixes to the docs or template files.
- New worked examples of `SOUL.md` / `IDENTITY.md` personalities.
- Adapters or notes for running the workspace on other runtimes.
- Stronger safety defaults.

## Ground rules

- **No personal data, ever.** Template files (`USER.md`, `TOOLS.md`, `MEMORY.md`) ship blank on purpose.
  Don't commit filled-in copies, real credentials, hostnames, or tokens.
- **Keep it runtime-agnostic** where possible. Runtime-specific notes belong in clearly labeled sections.
- **Small and readable beats clever.** The whole appeal is that you can read the agent's mind in a text editor.

## How

1. Fork, branch, change.
2. Open a PR describing *why* the change helps someone building an agent.

That's it. Thanks for helping make personal agents a little more legible.
